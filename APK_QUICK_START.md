# 📱 Android APK Quick Setup - ENDY EDESON Calculator

## 🚀 Fastest Path to APK (5 minutes)

### Step 1: Install Prerequisites
Download and install these (if not already installed):
- **Android SDK**: Part of Android Studio → https://developer.android.com/studio
- **Java JDK 11+**: https://www.oracle.com/java/technologies/downloads/
- **Node.js + npm**: https://nodejs.org/

### Step 2: Run Build Script (Automated)

**Windows:**
```bash
build-apk.bat
# Then choose option 1 (Capacitor)
```

**Linux/macOS:**
```bash
chmod +x build-apk.sh
./build-apk.sh
# Then choose option 1 (Capacitor)
```

### Step 3: Open in Android Studio
```bash
npx cap open android
```

### Step 4: Build APK
In Android Studio:
1. Click **Build** menu
2. Select **Build Bundle(s) / APK(s)**
3. Click **Build APK(s)**
4. Wait for build to complete (~2-3 minutes)

### Step 5: Get Your APK
Location: `android/app/build/outputs/apk/debug/app-debug.apk`

## 🔧 Manual Installation (No Build Tools)

If you don't have Android SDK installed:

1. **Download Android Studio**
   - https://developer.android.com/studio
   - Install with Android SDK (takes ~10-15 minutes)

2. **After Installation:**
   ```bash
   # Set environment variables
   ANDROID_HOME=/path/to/android/sdk
   JAVA_HOME=/path/to/jdk
   ```

3. **Create Virtual Device**
   - Android Studio → Tools → AVD Manager
   - Click "Create Virtual Device"
   - Choose device & Android version
   - Click Play to start emulator

4. **Run Build:**
   ```bash
   npm install
   npm install -g @capacitor/cli @capacitor/android
   npx cap init "ENDY EDESON Calculator" "com.endyedeson.calculator"
   npx cap add android
   npx cap sync
   npx cap open android
   ```

### Method 1: Direct File Installation
1. Download `.apk` file from build output
2. Send to Android phone (email, cloud, USB)
3. Open file manager on phone
4. Tap the APK file
5. Confirm installation

### Method 2: ADB (Command Line)
```bash

# Enable USB Debugging in Settings → Developer Options

```

### Method 3: Google Play Store (Publishing)
1. Create Google Play Developer account ($25)
2. Upload signed APK
3. Fill app store details
4. Submit for review (24-72 hours)

## 🔐 Create Signed Release APK

For Google Play or distribution:

### Option A: Android Studio GUI
1. Build → Build Bundle(s)/APK(s) → Build APK(s)
2. Click on "Release" variant
3. Android Studio guides you through signing
4. APK generated: `android/app/build/outputs/apk/release/app-release.apk`

### Option B: Command Line
```bash
# Generate keystore (one-time)
keytool -genkey -v -keystore my-release-key.keystore \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias calculator-key

# Build release APK with gradle
cd android
./gradlew bundleRelease
./gradlew assembleRelease
```

## 🐛 Troubleshooting

### "gradle not found"
```bash
# Download gradle wrapper
cd android
./gradlew --version
```

### "SDK not found"
```bash
# Set Android SDK path
export ANDROID_HOME=/path/to/android/sdk
export PATH=$PATH:$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools
```

### "Gradle build failed"
```bash
# Clean and rebuild
cd android
./gradlew clean
./gradlew build
```

### "APK installation fails on device"
1. Ensure Android 5.0+ (API 21+)
2. Enable "Unknown Sources" in Security
3. Check device storage space
4. Try debug APK first, not release

### "White screen on app start"
1. Check if `index.html` exists in assets
2. Verify file paths are correct
3. Check Android Studio logcat for errors
4. Ensure JavaScript is enabled in WebView

## 💡 Pro Tips

1. **Size Optimization**
   - Enable ProGuard/R8 in Gradle
   - Compress images to WebP format
   - Remove unused dependencies

2. **Performance**
   - Enable hardware acceleration
   - Use lazy loading for images
   - Minimize bundle size

3. **Distribution**
   - Test on multiple Android versions
   - Get real device testing (not just emulator)
   - Use Firebase for crash analytics

4. **Updates**
   - Version code must increment for each release
   - Use versioning: 1.0.0 → 1.0.1 → 1.1.0 → 2.0.0
